Versionsstyring med GIT

Versionsstyring

Versionsstyring er en god ide hvis man vil kunne samarbejde, undgå fejl i projektet, at kunne gå tilbage hvis de alligevel opstår og holde styr på flere delprojekter. Det kan være så enkelt som at holde styr på mapper og filer og kan suppleres med noter om hvad man ændrer. Softwaren kan både være en tjeneste ”i skyen”, eller det kan være installeret på en lokal server, hvor den holder styr på forskelle mellem de forskellige versioner.

Et af de meste udbredte metoder som webudviklere bruger er lavet på git-protokollen, en anden er SVN (subversion).

Man kan bruge versionsstyring til alle slags it-projekter og det er alle der kan bruge det, lige fra store webbureauer til Hr. Jensen der sidder og koder som en hobby i sin fritid. Man kan selvfølgelig også bruge det til andet end kildekode, f.eks. aftaler, rapporter osv.

Git

Git er en protokol oprindelig skrevet af Linus Thorvalds til at holde styr på udviklingen af Linux efterhånden som der kom flere med til at lave det.

Man kan selv lave en git-server eller man kan bruge en der ligger online, som f.eks. BitBucket eller GitHub.

GitHub

GitHub er et online versionsstyrings program hvor man, blandt andet, kan oprette repositories, samarbejde og lave projektstyring. Det er et socialt medie hvor man kan dele kode, og kan også bruges som et portfolio for dette. Der er også et desktop-program til GitHub som man kan downloade til sin computer – GitHub Desktop.

Det er en god ide når man bruger GitHub, at lave commits tidligt og ofte, og huske at lave kommentarer der fortæller hvilke ændringer man har lavet.

Github vejledning lavet sammen med Emilie Skov og Thea Winther-Nielsen

Forklaringer på udtryk i GitHub

  • Repository: (repo) en samling af filer, som en projektmappe.
  • Branch: En forgrening, en parallel version i dit repo. Berører ikke master.
  • Master branch: Hovedforløbet, den der gælder.
  • Commit: En revision, som at gemme, men med mulighed for at finde tilbage.
  • Pull request: Et forslag til en ny udgave (rettelser).
  • Diff: Difference, en forskel mellem to commits.
  • Merge: Når en ændring i en branch samles med en anden (eks. gennem en pull request).
  • Pull: Når du henter ændringer og merger dem.
